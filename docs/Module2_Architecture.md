# 模块二：量化编排引擎 - 架构设计文档

## 1. 设计目标
构建一个**绝对零阻塞**的量化编排系统，在进行高频率数据扫描和复杂指标计算时，确保不影响 OpenClaw 及其自身的主事件循环。

## 2. 核心架构

### 2.1 进程级隔离模型
```
主启动器 (Launcher)
    ├── 扫描器进程 (Scanner Worker) - 独立进程，同步读取数据库
    ├── 计算器进程 (Calculator Worker) - 独立进程，调用 Python 计算
    └── 警报器进程 (Alerter Worker) - 独立进程，异步发送通知
```

### 2.2 通信机制
- **IPC 消息传递**：主进程与子进程间使用 `process.send()` / `process.on('message')`
- **文件系统缓存**：计算结果写入 JSON 文件，避免进程间复杂数据传递
- **数据库状态表**：使用 SQLite 表记录任务状态和警报冷却期

## 3. 技术实现

### 3.1 主启动器 (Launcher)
**职责**：进程生命周期管理、健康监控、消息路由
**特点**：
- 绝对轻量，不执行任何耗时操作
- 自动重启崩溃的子进程（5秒超时检测）
- IPC 消息中心，路由数据流

### 3.2 扫描器进程 (Scanner)
**职责**：高效读取市场数据
**特点**：
- 独立进程，使用 `better-sqlite3` 同步 API
- 避免异步 I/O 的状态竞争问题
- 自动初始化数据表和状态表

### 3.3 计算器进程 (Calculator)
**职责**：指标计算与结果持久化
**特点**：
- 接收市场数据，触发 Python 指标计算
- 计算结果同步写入文件缓存
- 实时更新任务状态到数据库

### 3.4 警报器进程 (Alerter)
**职责**：异步消息推送
**特点**：
- 专门负责 Discord 消息推送
- 完全解耦于计算流程
- 支持消息格式化和优先级队列

## 4. 数据流设计

```
市场数据 → Scanner (同步读取) → Launcher (消息路由) 
    → Calculator (计算+缓存) → Launcher (状态更新)
    → Alerter (异步通知) → Discord
```

## 5. 性能保障

### 5.1 零阻塞保证
- **进程隔离**：所有耗时操作在独立进程中执行
- **同步读取**：Scanner 使用同步 API，避免回调地狱
- **异步通知**：Alerter 不等待推送响应

### 5.2 高可用性
- **自动重启**：子进程崩溃后 5 秒内自动恢复
- **状态持久化**：计算状态实时写入数据库
- **缓存降级**：Python 计算失败时使用最近缓存

### 5.3 可扩展性
- **插件化架构**：轻松添加新指标和信号类型
- **水平扩展**：可以增加更多 Calculator 进程
- **配置驱动**：所有参数通过配置文件管理

## 6. 目录结构

```
laicai/core/
├── src/
│   ├── launcher/           # 进程生命周期管理器
│   ├── scanner/            # SQLite 同步扫描器
│   ├── calculator/         # 指标计算与状态更新
│   ├── alerter/            # Discord 消息分发
│   ├── types/              # 跨进程消息与接口定义
│   └── common/             # 通用配置工具
├── data/                   # 数据存储目录
│   ├── cache/              # 计算缓存 (JSON 文件)
│   └── market.db           # SQLite 数据库
├── scripts/                # 测试和工具脚本
├── docs/                   # 设计文档
└── config/                 # 配置文件
```

## 7. 部署与监控

### 7.1 启动方式
```bash
# 一键启动
bash start.sh

# 或使用 PM2（推荐）
pm2 start ecosystem.config.js --name "laicai-orchestrator"
```

### 7.2 监控指标
- **进程健康**：子进程存活状态
- **计算延迟**：从扫描到计算的端到端延迟
- **缓存命中率**：文件缓存的使用效率
- **警报频率**：Discord 消息发送成功率

### 7.3 日志管理
- **结构化日志**：JSON 格式，便于分析
- **分级输出**：DEBUG/INFO/WARN/ERROR
- **日志轮转**：自动清理旧日志文件

## 8. 安全考虑

### 8.1 数据安全
- **敏感信息**：API 密钥等敏感配置不提交到版本库
- **访问控制**：数据库文件权限限制
- **加密传输**：Discord Webhook 使用 HTTPS

### 8.2 系统安全
- **进程沙箱**：子进程在受限环境中运行
- **资源限制**：限制 Python 进程的内存和 CPU 使用
- **输入验证**：所有输入数据都经过严格验证
