# LaiCai (来财) 量化投研助手 - 需求规格说明书 (PRD) V1.0

## 1. 项目愿景 (Vision)
构建一个基于 A/HK/US 全球资产的、**数据驱动且零幻觉**的智能投资助手。它不参与自动交易，但通过多维度的实时监控与量化审计，为用户提供极其精准的投资决策辅助。

## 2. 核心原则 (Core Principles)
1. **数据真实性第一 (Data Fidelity First)**: 拒绝 AI 幻觉，所有价格与指标必须来源于经过“双路校验”的真实行情。
2. **权责彻底隔离 (Separation of Concerns)**: 
    - AI 负责“投研、逻辑分析、信号生成”。
    - 独立引擎负责“数据抓取、计算、审计、入库”。
3. **架构高可扩展 (High Scalability)**: 支持从目前的 17 只标的平滑扩展至全市场扫描。

## 3. 功能需求 (Functional Requirements)

### 3.1 模块一：数据中心 (Data Engine)
- **多市场 K 线同步**: 自动补齐 A 股、港股、美股以及 ETF 的历史与实时 K 线（日线级）。
- **双向指针同步**: 具备 `Forward Sync` (补齐至今日) 与 `Backward Sync` (深挖至 IPO) 能力。
- **数据审计 (Auditor)**: 
    - **Gap Detection**: 自动对比交易所法定交易日历，识别并自动修复数据空洞。
    - **Sanity Check**: 逻辑自洽校验（如 High >= Low）。

### 3.2 模块二：量化编排层 (Orchestrator - Node.js)
- **指标实时计算**: 定期计算 RSI, 布林带 (BOLL) 等核心技术指标。
- **阈值警报系统**: 基于 `watchlist.json` 设定的买入/卖出区间，实时监控并触发警报。

### 3.3 模块三：AI 投研助手 (AI Analyst)
- **多维报告生成**: 基于本地数据库的真实数据，生成个股分析周报。
- **自然语言交互**: 用户可通过 Discord 询问个股的量化状态。

## 4. 非功能需求 (Non-Functional Requirements)
- **性能**: 核心同步引擎需支持并发处理，且具备 PID 锁和心跳容灾。
- **安全**: 敏感配置（API 密钥、个人持仓）严禁提交至公共仓库。
- **隔离性**: 编码任务需在独立沙箱 (ACP/Claude Code) 中执行，避免上下文过载。

## 5. 迭代规划 (Roadmap)
- **V1.0 (Current)**: 核心数据引擎、本地 SQLite 数据库、17 只标的深度同步、自动化审计。
- **V1.1**: Node.js 任务调度器、指标计算、Discord 信号警报。
- **V2.0**: 全市场异动扫描、AI 深度投研大脑。
